ReMap - UI Improved v3.34
==========================

ReMap - UI Improved v3.19
========================


이번 추가 수정 내용 - v3.34
--------------------
1. 학생 준비 화면 방 정보 배치 개선
   - 스마트폰 세로 화면에서 방 제목은 크게 유지하고, 방 코드/게임 모드/배틀당 문제 수/문제당 제한시간/맵 종류/닉네임을 3열 2행으로 정리했습니다.

2. 기본 총 게임 시간 변경
   - 교사 방 생성 화면의 총 게임 시간 기본값을 300초에서 180초로 변경했습니다.

3. 내 캐릭터 강조 방식 개선
   - 본인 캐릭터 뒤의 은은한 노란빛은 유지하고, 동그란 테두리 선은 제거했습니다.

4. 교사 PC 방 생성 화면 압축
   - 배틀 문제 수, 제한시간, 총 게임 시간 등 세부 설정 영역을 4열 배치로 정리하여 3행 안에 들어오도록 개선했습니다.

프로젝트 개요
------------
교사/학생 실시간 웹 게임 ReMap 실행 파일입니다.
교사가 방을 만들고 학생들이 방 코드로 입장한 뒤, 맵에서 이동하며 만나면 문제 배틀이 진행됩니다.
개인전/팀전, 미로형/오픈형 맵, 게임 시작 카운트다운, VS 배틀, 시상식 결과 화면을 지원합니다.


이번 추가 수정 내용 - v2.0
--------------------
1. 갤럭시/모바일 학생 문제 입력 화면 개선
   - 가로 화면에서 문제 입력 패널을 더 크게 보여주도록 조정했습니다.
   - 준비 화면이 화면 안에 들어오도록 카드 높이와 하단 버튼 영역을 다시 정리했습니다.
   - 계속 페이지 전체를 스크롤하지 않고 문제 입력 영역 안에서만 필요한 만큼 스크롤되도록 수정했습니다.

2. 모바일 학생 게임 화면 개선
   - 스마트폰에서는 PC용 좌우 정보 패널을 숨기고 게임 맵이 한 화면에 들어오도록 조정했습니다.
   - 갤럭시 가로 화면 기준으로 상단바와 맵 패널 높이를 줄였습니다.
   - 모바일용 방향 버튼을 추가했습니다.

3. Windows WebSocket 연결 종료 오류 완화
   - 학생 폰 회전, 브라우저 닫기, 네트워크 전환 때 나타날 수 있는 [WinError 121] 세마포 제한 시간 만료 메시지를 정상 연결 종료로 처리하도록 수정했습니다.
   - 실행 스크립트의 WebSocket ping timeout을 넉넉하게 조정하고 로그 출력을 줄였습니다.

이번 수정 내용
--------------
1. [다음 게임 준비] 후 두 번째 게임에서 이동 속도가 빨라지는 문제 수정
   - 학생 화면의 애니메이션/이동 루프가 게임을 다시 시작할 때 중복 실행되지 않도록 정리했습니다.
   - 게임 종료, 홈으로 이동, 다음 게임 준비, 소켓 연결 종료 시 기존 루프와 타이머를 안전하게 정리합니다.
   - 기존 이동속도 기본값 7은 유지했습니다.

2. 실행 안내 보강
   - CMD/터미널에서 실행해야 할 명령어를 아래에 정리했습니다.
   - 같은 와이파이/내부망 학생들이 접속할 수 있도록 실행 스크립트의 서버 host를 0.0.0.0으로 변경했습니다.

3. QR 입장 기능 추가
   - 교사 운영 화면의 학생 입장 코드 영역에 QR 코드가 함께 표시됩니다.
   - QR 코드는 학생 접속 주소와 방 코드를 포함합니다.
   - 학생이 스마트폰 카메라로 QR을 스캔하면 학생 화면이 열리고 방 코드가 자동 입력됩니다.

실행 전 준비
------------
1. Python 3.10 이상 설치를 권장합니다.
2. ZIP 파일을 압축 해제합니다.
3. CMD 또는 터미널을 열고 압축을 푼 폴더로 이동합니다.
   예시 Windows:
     cd C:\Users\사용자이름\Downloads\remap_ui_improved
   예시 Mac/Linux:
     cd ~/Downloads/remap_ui_improved
4. 처음 실행하는 컴퓨터라면 필요한 라이브러리를 설치합니다.
   Windows:
     python -m pip install -r requirements.txt
   Mac/Linux:
     python3 -m pip install -r requirements.txt

실행 방법 - Windows
-------------------
방법 A: run_server.bat 더블클릭

방법 B: CMD에서 직접 실행
  cd 압축을_푼_폴더\remap_ui_improved
  python -m uvicorn app.main:app --host 0.0.0.0 --port 8000

실행 방법 - Mac/Linux
---------------------
방법 A: 터미널에서 실행
  cd 압축을_푼_폴더/remap_ui_improved
  ./run_server.sh

방법 B: 직접 실행
  cd 압축을_푼_폴더/remap_ui_improved
  python3 -m uvicorn app.main:app --host 0.0.0.0 --port 8000

접속 주소
---------
서버를 실행한 교사용 컴퓨터에서 접속:
  교사 화면: http://127.0.0.1:8000/teacher
  학생 화면: http://127.0.0.1:8000/

학생들이 다른 기기에서 접속:
  http://교사용컴퓨터의_IP주소:8000/

스마트폰 접속:
  - 교사 화면의 QR 코드를 학생 스마트폰 카메라로 스캔합니다.
  - QR로 접속하면 방 코드가 자동 입력됩니다.
  - 학생은 닉네임을 입력하고 [다음]을 누르면 됩니다.

관리자도 스마트폰/태블릿에서 운영 가능:
  http://교사용컴퓨터의_IP주소:8000/teacher

예시: 교사용 컴퓨터 IP가 192.168.0.25라면
  교사 화면: http://192.168.0.25:8000/teacher
  학생 화면: http://192.168.0.25:8000/

교사용 컴퓨터 IP 확인 방법
-------------------------
Windows:
  1. CMD 실행
  2. ipconfig 입력
  3. Wi-Fi 또는 이더넷 항목의 IPv4 주소 확인
     예: 192.168.0.25

Mac:
  1. 터미널 실행
  2. ipconfig getifaddr en0 입력
  3. 출력된 주소 확인

Linux:
  1. 터미널 실행
  2. hostname -I 입력
  3. 출력된 주소 중 내부망 주소 확인

수업 중 사용 순서
----------------
1. 교사용 컴퓨터에서 서버 실행
2. 교사 화면 접속: http://127.0.0.1:8000/teacher 또는 http://교사용컴퓨터IP:8000/teacher
3. 교사가 방 설정 후 방 생성
4. 학생은 학생 화면 접속: http://교사용컴퓨터IP:8000/ 또는 교사 화면의 QR 코드 스캔
5. QR로 접속한 학생은 방 코드가 자동 입력되므로 닉네임 입력 후 [다음] 클릭
6. 학생은 팀/색상 선택 및 문제 제출
7. 교사가 [게임 시작] 클릭
8. 게임 종료 후 다시 할 때는 교사가 [다음 게임 준비] 클릭
9. 학생들은 준비 화면에서 다시 입장 완료 후 다음 게임 진행

주의 사항
---------
- 교사용 컴퓨터와 학생 기기는 같은 와이파이 또는 같은 내부망에 있어야 합니다.
- 학생 기기에서 접속이 안 되면 Windows 보안/방화벽에서 Python 또는 포트 8000 허용이 필요할 수 있습니다.
- QR이 열리지 않거나 localhost 주소로 보이면, 교사 화면을 http://교사용컴퓨터IP:8000/teacher 주소로 다시 열어 사용하세요.
- 학교 와이파이에 따라 기기 간 접속이 차단된 네트워크일 수 있습니다. 이 경우 같은 핫스팟이나 같은 내부망을 사용해야 합니다.
- 서버 실행 창은 게임 중 닫으면 안 됩니다. 창을 닫으면 접속이 끊깁니다.
- 포트 8000이 이미 사용 중이면 다음처럼 다른 포트를 사용할 수 있습니다.
  python -m uvicorn app.main:app --host 0.0.0.0 --port 8001
  이 경우 접속 주소도 :8001 로 바꿔야 합니다.

개발/수정용 참고
----------------
코드를 수정하면서 자동 재시작을 쓰고 싶다면 --reload를 붙여 실행할 수 있습니다.
  python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

[Render 무료 서버 배포 안내]
이 ZIP은 GitHub/Render 업로드용으로 정리된 버전입니다.

Render Web Service 설정:
- Build Command:
  pip install -r requirements.txt
- Start Command:
  uvicorn app.main:app --host 0.0.0.0 --port $PORT

배포 후 접속:
- 학생: https://서비스이름.onrender.com/

- 서버가 재시작되면 현재 진행 중인 방/게임 상태는 사라질 수 있습니다.

[다중 교사 독립 방 안내]
- 같은 Render 서버 주소를 여러 선생님이 함께 사용해도, 방 코드별로 방 상태가 분리됩니다.
- 교사 화면에서 방 생성 시 '방장' 이름을 입력하면 방 설정 요약에 방장 이름이 표시됩니다.
- 방을 만들면 교사용 주소가 /teacher?room=방코드 형태로 바뀌며, 이 주소는 해당 방을 관리하는 화면입니다.

- 현재 버전은 관리 비밀번호 없이 방 코드 기준으로 분리합니다. 교사용 관리 주소를 다른 사람과 공유하면 그 사람도 해당 방을 조작할 수 있습니다.
- Render 무료 서버가 재시작되거나 잠들었다가 새로 기동되면 메모리에 저장된 방 상태는 사라질 수 있습니다.

이번 수정 내용 - v3.29
--------------------
1. AI 문제 검토 기능 추가
   - 교사 운영 화면에 [AI 문제 검토] 버튼을 추가했습니다.
   - 학생이 문제를 제출한 뒤 교사가 선택적으로 눌러 학생 제출 문제의 정답/표현을 AI가 참고 검토할 수 있습니다.
   - 학생 제출과 게임 진행을 막지 않는 보조 검토 방식입니다.
   - 검토 결과는 교사용 화면에 '통과 / 확인 필요 / 오류 가능성 / 표현 모호 / 검토 불가'로 표시됩니다.

2. 엑셀 다운로드 보강
   - [문제 엑셀 다운로드] 파일에 AI 검토 결과, AI 의견, AI 추천답 열을 추가했습니다.
   - AI 검토를 실행한 뒤 다운로드하면 검토 결과가 함께 정리됩니다.

3. Gemini API 선택 연동
   - AI 검토를 사용하려면 실행 환경에 GEMINI_API_KEY를 설정해야 합니다.
   - Render에서는 Web Service > Environment 메뉴에서 GEMINI_API_KEY 값을 추가한 뒤 재배포하면 됩니다.
   - 선택적으로 GEMINI_MODEL 환경변수를 설정할 수 있습니다. 기본값은 gemini-2.5-flash입니다.
   - API 키가 없으면 프로그램은 계속 정상 작동하지만, AI 문제 검토 버튼에는 키 설정 안내가 표시됩니다.

주의:
- AI 검토는 교사를 돕는 참고 의견입니다. 최종 판단은 교사가 확인해야 합니다.
- v3.30부터는 AI가 여러 정답을 동시에 제시하거나 학교 위치, 지역구, 주소, 최신 정보처럼 외부 사실 확인이 필요한 문항을 만났을 때 무리하게 통과/오류로 단정하지 않고 확인 필요 또는 표현 모호로 표시하도록 보수적으로 조정했습니다.
- 학생 이름, 학번 등 개인정보를 문제에 넣지 않도록 안내하는 것이 좋습니다.
- 무료 API는 사용량 제한이 있어 많은 문항을 한 번에 검토하면 지연 또는 제한 오류가 발생할 수 있습니다.
